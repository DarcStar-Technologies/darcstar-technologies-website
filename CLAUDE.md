## Project

DarcStar Technologies marketing website ‚Äî SvelteKit (Svelte 5) on **Cloudflare Pages**. Still close to a fresh `sv create` scaffold; most of `src/routes` and `src/stories` is template/demo content.

Non-negotiables:

- Package manager is **pnpm** (pinned in `packageManager`; `.npmrc` sets `engine-strict=true`). Use `pnpm`, never `npm`.
- **Svelte 5 runes only** ‚Äî runes mode is forced project-wide. No legacy `export let`. See [docs/svelte.md](docs/svelte.md).

## Lessons

Durable gotchas ‚Äî keep to one line; link to the doc that carries the detail.

- Keep `@tailwindcss/forms`; Skeleton's form components depend on it. ‚Üí [styling](docs/styling.md)
- Site is **dark-only**: `data-mode="dark"` hardcoded on `<html>` in `app.html`, no toggle. Distinct from `data-theme` (palette). ‚Üí [styling](docs/styling.md)
- Palette is the **custom `darcstar` Skeleton theme** (`src/themes/darcstar.css`); regenerate ramps via `node scripts/gen-theme.mjs`, don't hand-edit. ‚Üí [styling](docs/styling.md)
- The RGB **color-charge triad has ONE source** ‚Äî the theme (`tertiary`/`secondary`/`primary` = R/G/B). `--charge-*` and `CosmicBackdrop`'s canvas both derive from `--color-*-500`; never re-type the hexes. ‚Üí [styling](docs/styling.md)
- **Brand type is self-hosted** (Space Grotesk / Inter / JetBrains Mono via Fontsource `@import` in `layout.css` ‚Äî CSS, not a JS import, or svelte-check chokes). Body/heading tokens live in `gen-theme.mjs`; mono/sans in `layout.css` `@theme`; bare `<h1>`‚Äì`<h6>` need the base-layer rule (Skeleton styles only `.h1`‚Äì`.h6`). ‚Üí [styling](docs/styling.md)
- **Every page's hero uses one pattern**: `eyebrow` ‚Üí `#helix-slot` (CosmicBackdrop centres the RGB helix there) ‚Üí heading+lede in a **`glass-panel`** below the helix. Match the homepage/`/about`; never a bare centered heading. ‚Üí [styling](docs/styling.md)
- **Raised frosted surfaces share a `@utility glass` base**; variants (`glass-nav`/`glass-card`/`glass-btn`) `@apply glass` + deltas. A Tailwind v4 `@apply` **inlines styles but never adds the applied class name** ‚Äî a `class="glass-card"` panel carries no `glass-panel` token, so `querySelectorAll('.glass-panel')` matched nothing and the sheen silently missed every panel (#108). Fix + rule: the sheen selects **structurally** ‚Äî `[class*="glass-"]:not(.glass-field):not(.glass-menu)` (`glass-sheen.ts`) ‚Äî but that's a **substring** match, so name new raised surfaces `glass-*` and don't reuse `glass-` in a non-raised token (or extend the `:not(...)`). ‚Üí [styling](docs/styling.md)
- `pnpm preview` runs the built worker on the real Workers runtime (Wrangler), not `vite preview`. ‚Üí [commands](docs/commands.md)
- Run `pnpm gen` after changing `wrangler.jsonc` bindings to refresh the `Env` type. ‚Üí [deployment](docs/deployment.md)
- **Prod and dev are SEPARATE Turso DBs, one per Cloudflare Worker.** Prod Worker (`main`‚Üídarcstar.tech) ‚Üí prod DB; a **separate preview Worker** (`[env.preview]` in `wrangler.jsonc`, non-prod branches, `wrangler versions upload --env preview`) ‚Üí dev DB; local `.env` ‚Üí dev DB. Each Worker holds its own `DATABASE_*` secret, so `getDb()` has **no** env logic ‚Äî isolation is at the secret layer. First-time split runbook + the preview Worker's own secrets are in [deployment](docs/deployment.md#environments--databases). ‚Üí [deployment](docs/deployment.md)
- Changing the DB schema? Run `pnpm db:generate` + commit the `drizzle/` migration ‚Äî the **`drizzle` CI check** (+ a pre-commit hook) regenerates offline and fails on drift, gating merge‚Üíprod. **Dev DB** applies via `pnpm db:push`; **prod DB** applies via the **`deploy-prod` GitHub Action** which `pnpm db:migrate`s **then** `wrangler deploy`s on push to `main` (`PROD_DATABASE_*` + `CLOUDFLARE_API_TOKEN`/`CLOUDFLARE_ACCOUNT_ID` secrets), so a failed migration **blocks the deploy** (migrate-before-deploy; Workers Builds does previews only, prod auto-deploy disabled) ‚Äî never `db:push` prod. An existing `db:push` prod DB needs a one-time `pnpm db:baseline` so migrate is forward-only (skips replaying `0000`). ‚Üí [deployment](docs/deployment.md)
- `checkJs` is **off** ‚Äî generated Paraglide `.js` clashes with the Cloudflare `Request` global; no hand-written `.js` exists, so `.js` bodies aren't checked. ‚Üí [i18n](docs/i18n.md)
- **All UI copy lives in Paraglide messages** (`messages/*.json`); the `local/no-raw-text` ESLint rule blocks hardcoded `.svelte` text/attrs (Wordmark exempt). `es` is untranslated placeholder ‚Üí non-base locales are `noindex` via `Seo.svelte`'s `TRANSLATED_LOCALES` flag; hreflang deferred until `es` is real. ‚Üí [i18n](docs/i18n.md)
- e2e builds + previews the Cloudflare bundle; match the vitest project to your filename (`client` / `server` / `storybook`). ‚Üí [commands](docs/commands.md)
- Use the Svelte MCP (list-sections ‚Üí get-documentation) for Svelte/SvelteKit questions; run svelte-autofixer on any Svelte you write. ‚Üí [svelte](docs/svelte.md)
- Page head (title/description/OG/Twitter) comes from one `<Seo>` component ‚Äî render it **once per page** in `+page.svelte`, never the layout (SvelteKit merges heads ‚Üí duplicate tags). OG card is `node scripts/gen-og.mjs` (Chromium rasterizes the brand to 1200√ó630 PNG). ‚Üí [seo](docs/seo.md)
- Contact CTAs open **one** modal (`ContactDialog`, rendered in the layout) via the `contactDialog` rune; it submits through a SvelteKit **remote `form`** (`contact.remote.ts`) ‚Üí `contact_submission` in Turso, with a honeypot + IP/time throttle, then **two fire-and-forget Resend emails** (`contact-notify.ts`, via `ctx.waitUntil` ‚Äî never fails the submission): a lead ‚Üí `info@` and a localized **submitter ack** (Paraglide `contact_ack_*`, RFC-3834 auto-reply headers, `allSettled` so a bounced ack never drops the lead). Needs `RESEND_API_KEY` + a verified Resend domain. `.remote.ts` may live anywhere except `$lib/server`. Run `pnpm db:push` after schema changes (the default apply path; `pnpm db:generate`+`db:migrate` write/apply a versioned `drizzle/` trail when wanted ‚Äî see [deployment](docs/deployment.md)). ‚Üí [contact](docs/contact.md)
- **Better Auth gates the `/admin` submissions view (#69)** ‚Äî email/password sign-in exists; **public sign-up is open but gated** (Turnstile + email verification, #96 PR2 ‚Äî and only ever mints a least-privileged `user`), so the only way to make an **operator/admin** is `pnpm admin:create` (then the roster UI). Sign-in is a **server form action** (`login/+page.server.ts`, shared `LoginForm`) that works **without JS** and forwards to `getAuth().handler()` so it stays on the router's **DB-backed rate limiter** (`rateLimit.storage: 'database'` ‚Üí `rate_limit` table; a direct `auth.api` call would skip it) and forwards the session `Set-Cookie` back. `handleBetterAuth` populates `locals.user` for `/api/auth/*` + `/admin` + `/login` (de-localized match) **or any request carrying a session cookie** (`getSessionCookie` ‚Äî header-only, no DB), but mounts the API only under `/api/auth/*`; anonymous visitors (no cookie) still skip the lookup (#48 win preserved). Run `pnpm db:push` (adds `rate_limit`) + `pnpm admin:create` before the admin area works. ‚Üí [auth](docs/auth.md)
- **The navbar reflects auth state** ‚Äî root `+layout.server.ts` exposes a minimal `page.data.user` (`{ email }` or null) **plus a separate `page.data.isStaff`** (not nested in `user`, so the admin/account layouts that override `user` can't shadow it; typed in `app.d.ts`), so `Header.svelte` swaps "Sign in" for **Sign out + a dashboard link** ‚Äî **Admin** for staff, **Account** for an end-user (#96). Global sign-out is `routes/logout/+server.ts` (native form POST ‚Üí `auth.api.signOut` ‚Üí `/`). `pnpm smoke:signin` asserts both nav states. ‚Üí [auth](docs/auth.md)
- **Operator-roster management is the Better Auth `admin` plugin** ‚Äî `/admin/users` (+`[id]`) does list/create/edit/role/reset-password/force-logout/disable/delete via no-JS `auth.api.*` form actions. **Three roles** (#95): `admin` (super ‚Äî roster + all messages), `operator` (reads + **deletes** submissions, no roster), `user` (end-user, own account only ‚Äî dormant until the #96 portal); `/admin` is gated by `isStaff` (admin **or** operator), not mere auth, and the submissions view has a staff-only `delete` action. Role helpers live in `admin-access.ts` (`ROLES`/`coerceRole`/`isStaff`/`apiRole`). **Owner bootstrap = `ADMIN_USER_IDS`** env allowlist (admin before any role check ‚Üí can't be locked out; `admin-access.ts` `isRosterAdmin`); it's a runtime var, so add to `.env` + `pnpm gen`, `wrangler secret put` in prod. The plugin is **schema-affecting** (adds `user.role/banned/‚Ä¶` + `session.impersonated_by`) ‚Üí bare `admin()` mirrored in `auth-cli.ts`, `pnpm db:push` adds the columns. Authz is **authoritative** (bypasses cookie-cache); UI self/owner guardrails aren't a hard boundary (raw `/api/auth/admin/*` has no owner concept). ‚Üí [auth](docs/auth.md)
- **Signed-in page views skip the DB** ‚Äî `session.cookieCache` (`auth-options.ts`, 5-min `maxAge`) serves the site-wide `getSession` from a signed `session_data` cookie snapshot, not a per-view DB read; sign-out clears the snapshot. Behavioral, NOT mirrored to the CLI config. **But** the hook resolves `/admin`+`/login`+`/account`+`/api/auth/*` **authoritatively** (`disableCookieCache`), so roster force-logout/disable are immediate there ‚Äî the cache only fronts the navbar (staleness is cosmetic). ‚Üí [auth](docs/auth.md)
- **End-user portal (`/account`, #96) activates the dormant `user` role** ‚Äî a self-service surface (own profile via `auth.api.updateUser`/`changePassword`, own messages) **separate from `/admin`**, gated only by sign-in (no `isStaff`). Message ownership: `contact_submission.user_id` (nullable FK, `set null`), linked via `linkSubmissionsToUser` (`contact-ownership.ts`) on signed-in submit, admin-create (vouch), or self-registered email-verify (PR 2). Every `/account` query is scoped to `locals.user.id` (the load-bearing isolation). Delivered in **two PRs**: PR 1 = ownership + portal; **PR 2 = public sign-up** (`/signup`) ‚Äî re-opens #48 behind **Cloudflare Turnstile** (`captcha` plugin, `endpoints:['/sign-up/email']` ONLY, or it breaks the no-JS `/login`) + **`requireEmailVerification`** (unverified sign-in ‚Üí 403; `afterEmailVerification` runs the ownership backfill). Staff-lockout guard: migration `0003_verify_existing_users` + roster-create/`admin:create` set `emailVerified` before it ships. Turnstile keys via `wrangler secret`; sign-up **requires JS** (widget via `{@attach}`). **#115** adds a **resend-verification** affordance ‚Äî on the signup "check your email" panel AND in the login form's "verify your email" state (`LoginForm.svelte`, shared by `/login` + the navbar `LoginDialog`; sibling `resend` actions in `signup/` + `login/+page.server.ts`) ‚Üí `POST /send-verification-email`, which is **already anti-enumerating** (constant-time, identical `{status:true}` for unverified/verified/absent, mails only a real unverified account) and is **NOT** in the captcha scope, so resend works **no-JS** ‚Äî don't add a widget or re-implement the generic response. ‚Üí [auth](docs/auth.md)
- **Every sign-in attempt is audited** ‚Äî a Better Auth **`hooks.after`** middleware (`auth-audit.ts`, wired in `auth.ts`) records success + failure to the **app-owned `login_audit`** table (`db/schema.ts` ‚Äî NOT a plugin table, so no `auth.schema.ts`/`auth-cli.ts` mirroring) **and** emits a structured `console.*` line ‚Üí Workers Logs. Captures email/raw-IP/UA/reason/status; links `userId` on success; never logs the password; returns `undefined` so anti-enumeration is intact. Fires for the `/login` form AND direct `/api/auth/sign-in/email`; the **one blind spot is rate-limit 429s** (rejected before endpoint dispatch) ‚Äî the login action records those. Persist is non-blocking (`waitUntil`, `login-audit-store.ts`, never fails sign-in). Read-only staff view at **`/admin/audit`**. ‚Üí [auth](docs/auth.md)
- **Self-service password reset** ‚Äî Better Auth's `forget-password` flow: `/forgot-password` (email ‚Üí anti-enumerating `POST /request-password-reset`, generic "check your email") ‚Üí emailed link ‚Üí `/reset-password?token=‚Ä¶` (new-password form ‚Üí `POST /reset-password`). Env-bound `sendResetPassword` + `revokeSessionsOnPasswordReset` live in `auth.ts` (email builder = `password-reset-email.ts`); **no schema change** (token in the existing `verification` table ‚Üí not in `auth-cli.ts`). Both pages are **single `default` actions** (never mix default+named, #122); no captcha (works no-JS); rate-limited (`/request-password-reset` 3/hr, `/reset-password` 10/hr). "Forgot password?" link is duplicated in `login/+page.svelte` + `LoginDialog.svelte`. ‚Üí [auth](docs/auth.md)
- **Sanity CMS content feed (DAR-14)** ‚Äî `/news` (post), `/research` (paper), `/people` (person) SSR-read the `production` dataset (project `8v6ikhvv`). Typed GROQ via `defineQuery`+**TypeGen** (`pnpm sanity:types` reads `src/lib/sanity/schema.json` ‚Äî **synced from `../darcstar-sanity-studio`**, NOT a local Studio ‚Äî + scans `queries.ts` ‚Üí committed `types.ts`; `overloadClientMethods` types `client.fetch(q)`). **Gotcha:** production is nominally _public_ but has **document-level access control** ‚Äî an anonymous read sees **only `siteSettings`**; post/paper/person need a **read token**, so reads go through the **server-only** `getSanityClient()` (`$lib/server/sanity.ts`, `readEnv('SANITY_VIEWER_TOKEN')`, `useCdn:false`), never the browser. Image URLs need no token (asset binaries are public by hashed URL) ‚Üí `image.ts` builds from `createImageUrlBuilder({projectId,dataset})`. `projectId`/`dataset`/`apiVersion` (`config.ts`) are **build-time** public config via `VITE_SANITY_*` (`import.meta.env`, inlined into BOTH bundles) with defaults ‚Äî NOT runtime `readEnv`, because the client builds image URLs that embed the dataset, so server + client must bake the SAME value (a runtime server-only read would desync). Set `VITE_SANITY_DATASET` in the build env to switch datasets. Portable Text via `@portabletext/svelte` (`PortableBody` + custom image/code/link). New secret ‚Üí `.env` + `.env.example` + `pnpm gen`, `wrangler secret put SANITY_VIEWER_TOKEN` (+ `--env preview`). ‚Üí [sanity](docs/sanity.md)
