# i18n — Paraglide (inlang)

The main non-obvious system in this repo.

- Messages live in `messages/{locale}.json` (`en`, `es`; base `en`), configured in `project.inlang/settings.json`. **All user-facing UI copy lives here** (issue #18) — components carry no hardcoded strings. Keys are flat snake_case with area prefixes (`hero_*`, `pillar_*`, `domain_*`, `footer_*`, `error_*`, `seo_*`, …). `es.json` currently holds **English placeholders** (Spanish is on the roadmap, not yet translated); Paraglide falls back to the base locale for any missing key, so a partial `es.json` never breaks the build.
- The Paraglide Vite plugin compiles them into `src/lib/paraglide/` (generated — do not edit). Import messages as `import { m } from '$lib/paraglide/messages.js'` and call `m.key({ ... })`; runtime helpers come from `$lib/paraglide/runtime`.
- Locale strategy is **`url`**. The pieces that wire this together:
  - `src/hooks.ts` (`reroute`) de-localizes the incoming URL so routing sees a single canonical path.
  - `src/hooks.server.ts` runs `paraglideMiddleware` and injects `%paraglide.lang%` / `%paraglide.dir%` into `src/app.html`.
  - `src/hooks.client.ts` instantiates `Locale` from `src/lib/paraglide.svelte.ts`, a runes class that overrides Paraglide's get/set-locale so setting a locale `goto`s the localized URL.
- To add a language: add the locale to `project.inlang/settings.json` and a `messages/<locale>.json`.

## Keeping copy out of components

A custom ESLint rule — [`eslint-rules/no-raw-text.js`](../eslint-rules/no-raw-text.js), wired in `eslint.config.js` as `local/no-raw-text` (**error**) over `src/routes/**` + `src/lib/components/**` — fails lint on hardcoded human text in `.svelte` templates and in `title`/`alt`/`aria-label`/`placeholder` attributes. It's a **forward guard**: it sees template text + those attributes, not copy hidden in JS string literals (data arrays, `{@render}` snippet args) — migrate those by hand. Dynamic `{m.*()}` and decorative glyphs (`·`, `→`, `©`, empty `alt`) are ignored; `Wordmark.svelte` is exempt (the "DarcStar Technologies" logotype is a proper noun, not translated).

## Untranslated `/es` — noindex until translated (issue #18)

`es` is wired but serves placeholder English, so `/es` is a duplicate. To keep it out of the index:

- [`Seo.svelte`](../src/lib/components/Seo.svelte) emits `<meta name="robots" content="noindex, follow">` for any non-base locale, keyed off a single `TRANSLATED_LOCALES = [baseLocale]` flag. That same flag gates `og:locale:alternate` (none emitted while `es` is placeholder).
- The old `display:none` per-locale `<a>` anchors in `+layout.svelte` (a poor SEO pattern) were removed.
- **When Spanish is really translated:** fill in `messages/es.json`, add `'es'` to `TRANSLATED_LOCALES`, and emit the reciprocal `<link rel="alternate" hreflang>` set (+ `x-default`) — a `TODO` in `Seo.svelte` marks the spot. hreflang is deferred until then because pointing it at a `noindex` page is contradictory.

**Lesson: `checkJs` is off** (`tsconfig.json`). Paraglide's generated `server.js` does `new Request(request.clone())`, which doesn't type-check against the Cloudflare runtime `Request` global that `worker-configuration.d.ts` injects (its `clone()` return type isn't assignable to its own constructor — harmless at runtime). Both files are generated, so the fix lives in `tsconfig.json`: since there's no hand-written `.js` in the app, we don't type-check `.js` bodies. Importers still get full type safety against Paraglide's inferred API.
